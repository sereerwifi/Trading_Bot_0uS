## CRITICAL (5 findings)

**1. harmonic_patterns.py:296 — Harmonic strategy has been voting the wrong direction on every signal**
`direction = "bullish" if x_type == "high" else "bearish"` is backwards. In Carney XABCD convention, X=HIGH means D is a resistance zone and the trade is SHORT. X=LOW means D is a support zone and the trade is LONG. Every harmonic signal ever fired has been the wrong way.
Fix: `direction = "bullish" if x_type == "low" else "bearish"`.

**2. harmonic_patterns.py:229 — `d_from_cd` formula projects D on the wrong side of C, destroying PRZ convergence**
`d_from_cd = C + cd_mid * (C - B)` always sends D in the wrong direction from C relative to the swing structure, so the two PRZ estimates (`d_from_xd` and `d_from_cd`) always diverge. `tightness` scores near 0 on every pattern.
Fix: change `(C - B)` to `(B - C)`.

**3. harmonic_patterns.py:244/313 — Fib zone cross-check uses the wrong zone type**
Because direction is inverted (finding #1), patterns near a resistance are cross-checked against support zones and vice versa — the bonus is essentially random. Fixed automatically once finding #1 is corrected.

**4. xauusd_mt5_strategy.py:1527 — `timedelta` not imported → NameError on every manual-trade close**
`from datetime import datetime, time as dtime` never imports `timedelta`, but `track_manual_trades()` calls `datetime.now() - timedelta(days=1)`. Every closed manual position triggers this NameError, swallowed by the main loop's outer except — the trade is silently dropped from `manual_trades.json` and the dashboard P&L.

**5. xauusd_mt5_strategy.py:2258/2548/3045 — `mt5.account_info()` used without None guard in 3 places**
`run_confluence_scan()`, `run_logic_groups_scan()`, and `run_once()` all call `info = mt5.account_info()` then immediately pass `info.balance` to lot-sizing/drawdown functions. A transient MT5 disconnect returns None → AttributeError → increments `consecutive_errors` → can trigger STOP_ON_ERROR.
